## Other Engineering

### Rust/C++ Interop Problem Space Mapping

The Rust Foundation has a new contractor working on the Rust/C++ Interop Initiative, particularly the all-important problem space mapping [project goal](https://github.com/rust-lang/rust-project-goals/issues/388).

They have [hit the ground running](https://github.com/rust-lang/rust-project-goals/issues/388#issuecomment-3970777014) and has developed a framework for specifying interop problems and many examples of current problems.

### crates.io Svelte Implementation

Work continues on migrating crates.io from Ember to Svelte.

Authentication has been [implemented](https://github.com/rust-lang/crates.io/pull/12877), porting the Ember `session` service to Svelte 5 using `createContext` for dependency injection and `$state` runes for reactive state tracking.

The API token settings page has been [implemented](https://github.com/rust-lang/crates.io/pull/13063) as well, based on the corresponding implementation in the Ember.js crates.io app.

### Malicious Crates Policy Update

There has been an update to the malicious crate notification policy.

The crates.io team will no longer publish a blog post each time a malicious crate is detected or reported. In the vast majority of cases to date, these notifications have involved crates that have no evidence of real world usage, and they feel that publishing these blog posts is generating noise, rather than signal.

They will always publish a [RustSec](https://rustsec.org/) advisory when a crate is removed for containing malware. Crates that contain malware and are seeing real usage or exploitation will still get both a blog post and a RustSec advisory.

### Reigning in Exploding Bandwidth Growth

We detected various attempts of abusing the crates-io API, which degrade the performance of the app and increase the amount of cloud credits required to keep it running.

We started experimenting with the Fastly WAF to enforce crates-io rate limits and block potential security threats. We are now evaluating the impact of this change to eventually applying it to 100% of the traffic.

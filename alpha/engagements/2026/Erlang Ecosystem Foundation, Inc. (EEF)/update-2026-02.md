## Published CVEs

### CVE-2026-23939: Path Traversal in Local File Store Backend

A path traversal vulnerability was discovered in the `Hexpm.Store.Local` module.
The affected routines (`get/3`, `put/4`, `delete/2`, and `delete_many/2`)
improperly validated pathnames, allowing relative path traversal attacks.

**Note:** This issue does NOT affect hex.pm the service. Only self-hosted
deployments using the Local Storage backend are affected.

* **Severity:** Medium (CVSS 4.0: 6.9)
* **Weakness:** CWE-22 (Improper Limitation of Pathname to Restricted Directory)
* [CVE-2026-23939](https://cna.erlef.org/cves/CVE-2026-23939.html)
* [hexpm/hexpm@5d2ccd2](https://github.com/hexpm/hexpm/commit/5d2ccd2f14f45a63225a73fb5b1c937baf36fdc0)

### CVE-2026-21619: Unsafe Deserialization of Erlang Terms

A vulnerability involving unsafe deserialization was discovered in `hex_core`.
This allows an attacker to inject arbitrary atoms into the Erlang VM, which can
lead to a denial of service by exhausting the atom table. Exploitation requires
the ability to control file contents, for example through CI cache poisoning or
by posing as hex.pm to deliver malicious content.

* **Severity:** Low (CVSS 4.0: 2.0)
* **Weakness:** CWE-400 (Uncontrolled Resource Consumption)
* **Affected:** hex_core (<0.12.1), hex (<2.3.2), rebar3 (<3.27.0)
* [CVE-2026-21619](https://cna.erlef.org/cves/CVE-2026-21619.html)
* [hexpm/hex@a9999e4](https://github.com/hexpm/hex/commit/a9999e425d9c0f9fc8c53e7823eee898156c4d1a)

## Additional Security Improvements

Beyond CVE remediation, we made several proactive security enhancements this
month:

### Content Security Policy Implementation

Implemented strict Content Security Policy with nonce-based script and style
protection for hex.pm, including:

* `script-src: 'strict-dynamic'` with nonces
* `style-src` with nonces
* Restrictive `frame-src`, `connect-src`, `object-src`, and `base-uri` policies

* [hexpm/hexpm#1385](https://github.com/hexpm/hexpm/pull/1385)

### GitHub Actions Security Hardening

Added comprehensive security scanning for CI/CD pipelines across repositories:

* CodeQL workflow for static analysis of GitHub Actions
* zizmor audit for detecting security vulnerabilities in CI/CD configuration
* Explicit permissions and `persist-credentials: false` on checkout actions
* Pinned GitHub Action versions
* Added Dependabot for automated dependency updates

* [hexpm/hex#1110](https://github.com/hexpm/hex/pull/1110)
* [hexpm/hexpm#1383](https://github.com/hexpm/hexpm/pull/1383)

### Safe Deserialization Best Practices

Replaced `:erlang.binary_to_term` with `safe_binary_to_term!` when parsing hex
package manifests to restrict deserialization to safe terms only.

* [hexpm/hex#1111](https://github.com/hexpm/hex/pull/1111)

### Frontend Dependency Updates

Updated frontend dependencies to address a jQuery version with a known CVE:

* Updated all npm packages to latest minor/patch versions
* Added Dependabot for automated npm dependency updates

* [hexpm/hexpm#1384](https://github.com/hexpm/hexpm/pull/1384)

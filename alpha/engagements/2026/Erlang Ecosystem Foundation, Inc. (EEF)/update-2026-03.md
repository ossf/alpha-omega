# Erlang Ecosystem Foundation Update — March 2026

## Hex.pm Security Audit Remediation Nearing Completion

Remediation work for the security audit findings is nearly complete. The majority
of issues have been resolved and re-testing has been completed. Remaining work
focuses on larger initiatives: security documentation/threat modeling and improved
isolation for hexdocs.pm.

### Rebar3 OAuth2 Support

Remediation was completed under budget. As planned in the grant proposal, the
unused remediation reserve is being redirected to additional Hex account-security
controls: specifically, implementing OAuth2 Device Grant authentication for
rebar3. This will enable deprecation of basic authentication and mandatory 2FA
across all package manager clients (Elixir, Gleam, and Erlang).

## Published CVEs

Three additional CVEs have been published this month:

### CVE-2026-21621: Read-Only API Key Privilege Escalation

A read-only API key could be elevated to full access when combined with a valid
2FA code, bypassing the intended permission restrictions.

* **Severity:** Medium
* **Weakness:** CWE-863 (Incorrect Authorization)
* [CVE-2026-21621](https://cna.erlef.org/cves/CVE-2026-21621.html)
* [GHSA-739m-8727-j6w3](https://github.com/hexpm/hexpm/security/advisories/GHSA-739m-8727-j6w3)

### CVE-2026-21622: Password Reset Link Never Expires

Password reset tokens did not expire, allowing indefinite use of reset links.

* **Severity:** Low
* **Weakness:** CWE-640 (Weak Password Recovery Mechanism)
* [CVE-2026-21622](https://cna.erlef.org/cves/CVE-2026-21622.html)
* [GHSA-6r94-pvwf-mxqm](https://github.com/hexpm/hexpm/security/advisories/GHSA-6r94-pvwf-mxqm)

### CVE-2026-23940: Library File Upload Denial of Service

A vulnerability in library file upload processing could cause denial of service.

* **Severity:** Medium
* **Weakness:** CWE-400 (Uncontrolled Resource Consumption)
* [CVE-2026-23940](https://cna.erlef.org/cves/CVE-2026-23940.html)
* [GHSA-jp8w-gxf6-8hcr](https://github.com/hexpm/hexpm/security/advisories/GHSA-jp8w-gxf6-8hcr)
* [hexpm/hex_core#165](https://github.com/hexpm/hex_core/pull/165)
* [hexpm/hex_core#166](https://github.com/hexpm/hex_core/pull/166)
* [hexpm/hex_core#167](https://github.com/hexpm/hex_core/pull/167)

## Additional Security Improvements

### Sudo Mode for Sensitive Operations

Implemented "Sudo Mode" requiring re-authentication for sensitive operations,
tightening protections around 2FA enrollment, recovery key access, and session
handling.

* [hexpm/hexpm#1402](https://github.com/hexpm/hexpm/pull/1402)
* [hexpm/hexpm#1403](https://github.com/hexpm/hexpm/pull/1403)

### OAuth2 Device Authorization Hardening

Removed the `verified=true` GET parameter from the OAuth2 device authorization
flow, preventing potential bypass of user verification.

* [hexpm/hexpm#1407](https://github.com/hexpm/hexpm/pull/1407)

### URL Scheme Restrictions

Short URLs are now restricted to http/https schemes only, preventing potential
abuse via other URI schemes.

* [hexpm/hexpm#1414](https://github.com/hexpm/hexpm/pull/1414)

### TLS Hardening

Disabled TLS 1.0 and TLS 1.1 support to enforce modern cryptographic standards.

### HexDocs Security Improvements

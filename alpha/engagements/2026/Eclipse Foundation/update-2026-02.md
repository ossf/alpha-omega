Work on the automated tooling continued, with design modifications, improvements, and scaffolding updates to streamline review data collection:

# Open VSX

### Malware & Vulnerability Scanning

* **Production Security Scanning:** Security scanning was officially enabled for the Open VSX production environment ([PR \#8188](https://github.com/EclipseFdn/open-vsx.org/pull/8188)).  
* **Active Scanners:** Both Yara and ClamAV scanning services were deployed in the Helm charts and subsequently activated within the application configuration ([PR \#8188](https://github.com/EclipseFdn/open-vsx.org/pull/8188)).  
* **Resource Optimization:** Resource limits for the ClamAV service were explicitly increased to ensure the reliability and stability of malicious file scans ([PR \#8213](https://github.com/EclipseFdn/open-vsx.org/pull/8213)).

### Supply Chain & Dependency Security

* **Webpack SSRF & Bypass Vulnerability:** The webpack dependency in the website repository was updated from version 5.94.0 to 5.104.1 ([PR \#7970](https://github.com/EclipseFdn/open-vsx.org/pull/7970)). This update patches a critical user information bypass vulnerability located within the HttpUriPlugin.  
* **Strict URI Redirection:** The patched Webpack version implements strict re-validation for redirects against allowed URIs, restricting them exclusively to HTTP/HTTPS. It also introduces a conservative redirect limit to effectively prevent Server-Side Request Forgery (SSRF) and untrusted content inclusion.  
* **GitHub Actions Security Fix:** The GitHub Actions dependencies were bumped, notably upgrading @actions/cache to v5.0.3/v5.0.5 to resolve a documented security vulnerability flagged by Dependabot ([PR \#8088](https://github.com/EclipseFdn/open-vsx.org/pull/8088)).

### Rate Limiting & Traffic Control

* **Dynamic Rate Limiting:** Static rate limiting was deprecated and disabled in favor of a newly implemented dynamic rate-limiting system ([PR \#8213](https://github.com/EclipseFdn/open-vsx.org/pull/8213)).  
* **Safety Tiers:** A baseline safety tier capping requests at 75 Requests Per Second (RPS) was established directly in the database to mitigate abuse, with future plans to adjust limits dynamically via the admin dashboard ([PR \#8213](https://github.com/EclipseFdn/open-vsx.org/pull/8213)).

### Application Security Enhancements

* **Version Obfuscation:** The application version number was removed from the HTTP header, reducing the surface area for targeted, version-specific exploits ([PR \#7926](https://github.com/EclipseFdn/open-vsx.org/pull/7926)).  
* **Temporary File Cleanup:** Deployment version 0.32.3 introduced a backend fix ensuring that all temporary extension files are strictly deleted during the publication process, preventing sensitive data or stale files from remaining on the server ([PR \#8225](https://github.com/EclipseFdn/open-vsx.org/pull/8225)).

### Monitoring & Threat Detection

* **Monitoring Dashboard:** A new Grafana dashboard was implemented specifically for tracking and visualizing .vsix download activity ([PR \#8425](https://github.com/EclipseFdn/open-vsx.org/pull/8425)).  
* **Anomaly Detection:** Alongside total stats and trend monitoring, the new dashboard includes dedicated "spike detection" panels, enhancing the operations team's ability to rapidly identify anomalous download behaviors, automated scraping, or potential Denial of Service patterns ([PR \#8425](https://github.com/EclipseFdn/open-vsx.org/pull/8425)).
